1. Objetivos del Módulo Auth
                                 * Permitir que alumnos y docentes inicien sesión de manera segura.
                                 * Facilitar el registro de nuevos usuarios con nombre y apellido por separado.
                                 * Redirigir a los usuarios a sus paneles correspondientes después del login.
________________


2. Flujos de Usuario
A. Flujo de Login
                                 1. Pantalla de Login:

                                    * Campos para correo y contraseña.
                                    * Opción para recuperar contraseña.
                                    * Botón para iniciar sesión.
                                    * Enlace para registrar una nueva cuenta.
                                    2. Validación:

                                       * Verificar que los campos no estén vacíos.
                                       * Validar credenciales contra la base de datos.
                                       * Mostrar mensajes de error si:
                                       * Credenciales incorrectas.
                                       * Campos vacíos.
                                       3. Redirección:

                                          * Si las credenciales son correctas, redirigir según el rol:
                                          * Alumno → Panel principal del alumno.
                                          * Docente → Panel principal del docente.
________________


B. Flujo de Registro (Signup)
B.1. Registro de Alumnos
                                          1. Pantalla de Registro:

                                             * Campos para:
                                             * Nombre.
                                             * Apellido.
                                             * Correo electrónico.
                                             * Contraseña (con confirmación).
                                             * Grado (ej: 10mo grado).
                                             * Sección (ej: A, B, C).
                                             2. Validación:

                                                * Correo válido y único.
                                                * Contraseña segura (mínimo 8 caracteres).
                                                * Confirmación de contraseña coincide.
                                                * Grado y sección deben ser seleccionados.
                                                3. Creación de Cuenta:

                                                   * Guardar los datos en la base de datos.
                                                   * Redirigir al login o al panel del alumno.
________________


B.2. Registro de Docentes
                                                   1. Pantalla de Registro:

                                                      * Campos para:
                                                      * Nombre.
                                                      * Apellido.
                                                      * Correo electrónico.
                                                      * Contraseña (con confirmación).
                                                      * Grados y secciones que imparte (selección múltiple).
                                                      2. Validación:

                                                         * Correo válido y único.
                                                         * Contraseña segura.
                                                         * Confirmación de contraseña coincide.
                                                         * Al menos un grado y una sección deben ser seleccionados.
                                                         3. Creación de Cuenta:

                                                            * Guardar los datos en la base de datos.
                                                            * Redirigir al login o al panel del docente.

4. Modelado de Datos
A. Tabla users
                                                            * id: Identificador único.
                                                            * nombre: Nombre del usuario.
                                                            * apellido: Apellido del usuario.
                                                            * correo: Correo electrónico.
                                                            * contraseña: Contraseña hasheada.
                                                            * rol: "alumno" o "docente".
                                                            * grado: Grado del alumno o docente (ej: "10mo").
                                                            * seccion: Sección del alumno o docente (ej: "A").
                                                            * created_at: Fecha de creación.
B. Tabla docentes_grados (solo para docentes)
                                                            * docente_id: ID del docente.
                                                            * grado: Grado que imparte (ej: "1ro").
                                                            * seccion: Sección que imparte (ej: "A").
________________


5. Requisitos Técnicos
A. Frontend
                                                            * Tecnologías: React + TypeScript.
                                                            * Componentes:
                                                            * Formularios para login y signup.
                                                            * Validación de campos en tiempo real.
                                                            * Mensajes de error/éxito.
                                                            * Redirección según el rol.
B. Backend
                                                            * Endpoints:
                                                            * POST /auth/login: Recibe credenciales y devuelve un token (JWT) y el rol.
                                                            * POST /auth/signup: Crea un nuevo usuario.
                                                            * POST /auth/forgot-password: Envía enlace de recuperación.
                                                            * POST /docentes/{docente_id}/grados: Asocia grados y secciones a un docente.
________________


6. Diagramas de Flujo
A. Login
                                                            1. Usuario ingresa credenciales → Validación → Redirige según rol.
B. Registro de Alumnos
                                                            1. Alumno ingresa datos → Validación → Guarda en BD → Redirige.
C. Registro de Docentes
                                                            1. Docente ingresa datos y selecciona grados/secciones → Validación → Guarda en BD → Redirige.

1. Colección users
Descripción: Almacena usuarios (docentes y alumnos) con datos esenciales para autenticación.
{
  _id: ObjectId,
  nombre: String,          // Nombre completo
  correo: {                // Único y obligatorio
    type: String,
    unique: true,
    required: true
  },
  contraseña: String,      // Encriptada (ej: bcrypt)
  rol: {                   // "docente" o "alumno"
    type: String,
    enum: ["docente", "alumno"],
    required: true
  },
  especialidad: String,    // Solo para docentes (ej: "Matemáticas")
  grado: String,           // Solo para alumnos (ej: "10mo A")
  estado: {                // Estado de la cuenta
    type: String,
    enum: ["activo", "pendiente", "inactivo"],
    default: "pendiente"
  },
  creadoEn: Date           // Fecha de creación
}
Ejemplo de documentos:
Docente:

  "rol": "docente",
  "especialidad": "Matemáticas",
  "estado": "activo",
  "creadoEn": ISODate("2023-10-15T09:30:00Z")
}
Alumno:

2. Colección requests
Descripción: Solicitudes de asociación entre alumnos y docentes.
{
  _id: ObjectId,
  alumnoId: {              // Referencia al alumno
    type: ObjectId,
    ref: "User",
    required: true
  },
  docenteId: {             // Referencia al docente
    type: ObjectId,
    ref: "User",
    required: true
  },
  mensaje: String,         // Mensaje opcional del alumno
  estado: {                // "pendiente", "aceptada", "rechazada"
    type: String,
    enum: ["pendiente", "aceptada", "rechazada"],
    default: "pendiente"
  },
  creadaEn: Date           // Fecha de creación
}

  destinatarios: [{        // Lista de grupos o "todos"
    tipo: { type: String, enum: ["grupo", "todos"] },
    id: { type: ObjectId, refPath: "destinatarios.tipo" }
  }],
  creadoEn: Date
}

10. Colección historial
Descripción: Historial de actividad de los usuarios (acciones importantes).
{
  _id: ObjectId,
  usuarioId: {             // Referencia al usuario
    type: ObjectId,
    ref: "User",
    required: true
  },
  accion: String,          // Descripción de la acción
  tipo: {                  // Tipo de acción
    type: String,
    enum: ["asignación", "respuesta", "perfil", "solicitud"],
    required: true
  },
  referenciaId: ObjectId,  // ID de referencia (ej: conjunto, pregunta)
  creadoEn: Date
}
Ejemplo:
{
  "_id": ObjectId("7f8d0d55b54764421b7156c3"),
  "usuarioId": ObjectId("5f8d0d55b54764421b7156a2"), // Juan Pérez
  "accion": "Completó la pregunta: Resuelve la ecuación: 2x + 3 = 7",
  "tipo": "respuesta",
  "referenciaId": ObjectId("6f8d0d55b54764421b7156b2"), // Pregunta 1
  "creadoEn": ISODate("2023-11-06T10:20:00Z")
}

11. Colección plantillas
Descripción: Plantillas para preguntas frecuentes o anuncios.
{
  _id: ObjectId,
  nombre: String,          // Nombre de la plantilla
  tipo: {                  // Tipo de plantilla
    type: String,
    enum: ["pregunta", "anuncio"],
    required: true
  },
  contenido: String,       // Contenido de la plantilla
  docenteId: {             // Referencia al docente
    type: ObjectId,
    ref: "User",
    required: true
  },
  creadoEn: Date
}
Ejemplo:
{
  "_id": ObjectId("7f8d0d55b54764421b7156c4"),
  "nombre": "Recordatorio de tarea",
  "tipo": "anuncio",
  "contenido": "Recuerden completar el conjunto {nombre_conjunto} antes del {fecha_límite}.",
  "docenteId": ObjectId("5f8d0d55b54764421b7156a1"), // Carlos Ruiz
  "creadoEn": ISODate("2023-10-25T11:45:00Z")
}
